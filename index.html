Summary

This project began as a basic automated robotic-arm segregator that would identify objects by color and place them into different locations. As the quarter went on, the scope of the project changed significantly. Instead of only making the robot do a simple pick-and-place sorting task, our team expanded the project into a robotic arm that can play tic-tac-toe against a human player. The robot uses the QArm, along with its RGB camera, to view the board, detect the game pieces, and make moves based on the state of the game. The human player uses blue 3D printed character pieces, while the robot uses gold pieces modeled after Dr. Mohammad Hasan. Overall, the final project became more challenging because it combines robotic movement, camera-based detection, 3D printed game pieces, and game decision-making into one system.

Motivation

Our original motivation was to build a robotic arm that could autonomously sort colored objects from different locations. While that idea was a good starting point, it was also somewhat limited in complexity. After discussing the project with Dr. Hasan, it became clear that the original segregator concept was too simple for the goals of the course. Because of that, our team started brainstorming ways to make better use of the robotic arm and its RGB camera.

We decided that tic-tac-toe would be a better application because it still involves pick-and-place motion, but it also adds vision, position detection, and decision-making. The robot does not just move random objects anymore. It has to recognize the board, understand where pieces are located, and place a game piece in the correct square without disturbing the others. This made the project more interesting and more representative of a real mechatronics system.

Another motivation for this change was that we had already 3D printed human character pieces during the early segregator phase. Instead of throwing away that work, we decided to keep the characters and turn them into tic-tac-toe tokens. This helped connect the old project idea with the new one. The blue character pieces meant for the human player represent members of our team, while the robot uses gold pieces modeled after Dr. Hasan, giving the project a more personal and lighthearted theme.

Gold character pieces in the staging area. These printed tokens helped carry the original segregator idea into the final tic-tac-toe version of the project.

Background

At the beginning of the project, our team planned to create a robotic color segregator. The goal was for the arm to detect objects by color and move them into separate locations. To test that idea, we designed and 3D printed small human-shaped pieces in different colors. These pieces were originally meant to act as sample objects for the sorting process. The first version of the project mostly focused on the basic robotic functions, such as arm movement, pick-and-place actions, and color detection.

However, once we received feedback that the project needed more depth, we changed direction. Instead of just sorting pieces by color, we decided to use the same robotic and vision concepts in a more advanced way. The project became a tic-tac-toe robot that uses camera input to view the board and identify positions, then uses the robotic arm to place a piece in the correct square. This new version still uses color detection and object handling, but now those features support a larger system involving board recognition, game logic, and more precise arm movement.

This project also builds on multiple areas of mechatronics. Mechanically, the arm must be able to move accurately within its workspace and place pieces without hitting others. From a controls standpoint, the robot must move smoothly and repeatably to known board coordinates. From a vision standpoint, the RGB camera is used to detect pieces and help determine the state of the game. Finally, from a programming standpoint, the system must combine movement control, image processing, and a tic-tac-toe decision algorithm. Because of these combined elements, the updated project is much more advanced than the initial segregator idea and better reflects the type of engineering problem-solving expected in this course.

Why this version is stronger

Compared with the original sorting concept, tic-tac-toe introduced more mechatronics depth. The robot had to perceive board state, solve game logic, and execute precise pick-and-place operations without disturbing existing pieces. That made the final design more interactive and much better aligned with the course goals.

The report emphasizes the project as a blend of mechanics, controls, computer vision, and programming rather than a single-function pick-and-place demo.

System overview

Robot platform

The QArm provides a 4 degree-of-freedom roll-pitch-pitch-roll arm with a gripper and RGB-D camera, which is a strong fit for repeatable grasping and placement.

Vision stack

OpenCV-based color masking, contour detection, and centroid extraction were used to detect tokens and infer the board state from the RGB camera feed.

Decision logic

A minimax-based game algorithm evaluates the board and selects the robot's next move while helper functions update moves, detect winners, and handle draws.
